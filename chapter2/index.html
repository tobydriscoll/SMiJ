using CairoMakie
"p3 - band-limited interpolation"
function p3()
    h = 1
    xmax = 10
    x = -xmax:h:xmax                     # computational grid
    xx = -xmax-h/20:h/10:xmax+h/20       # plotting grid
    sinc(t) = iszero(t) ? 1.0 : sinpi(t / h) / (π * t / h)
    funs = [
        x -> float(x==0),
        x -> float(abs(x) <= 3),
        x -> max(0, 1 - abs(x)/3)
        ]
    fig = Figure()
    for (i, f) in enumerate(funs)
        v = f.(x)
        ax = Axis(fig[i, 1], yticks=[0, 1], limits=(-xmax, xmax, -0.5, 1.5))
        scatter!(ax, x, v)
        p = sum( v[j] * sinc.(xx .- x[j]) for j in eachindex(x) )
        lines!(ax, xx, p)
    end
    return fig
end

Caution

Once a name is assigned to a function, it cannot be used as a variable name in the same scope.

In lines 10–12 I use an anonymous function (i.e., lambda) syntax

Ternary operator

The ternary operator in line 8

p ? A : B

evaluates to A if p is true; otherwise, B. This device is best used with short expressions, to keep the logic readable at a glance. Some style guides recommend against it entirely.

sinpi

The sinpi, cospi, and tanpi functions not only save a multiplication by π\piπ, but they can be more accurate, too.

Numeric type conversion

In lines 10 and 11 I use float to convert the logical results of type Bool to floating-point numbers. The explicit conversions aren’t strictly necessary here, but they improve clarity. A more general syntax for type conversion is convert(T, x), which converts x to type T.

max vs. maximum

From line 12, note that max chooses the largest of its arguments. This differs from previously-seen maximum, which returns the largest element of an array.

Subplots

Line 17 shows how to use indexing of a figure object to create subplots. We’ll be seeing a lot of that as we go. You can even have grids within subplot grids to construct complicated layouts.

Generators

Line 19 contains more than it might seem. The goal is to evaluate a truncation of the sum in (2.10),

∑mvmSh(x−xm),\sum_{m} v_m S_h(x - x_m),m∑​vm​Sh​(x−xm​),

for all values of xxx in the plotting range xx. The approach taken here is to evaluate each term at all values of xxx and sum the resulting vectors. While the syntax resembles a comprehension, it is instead a more primitive generator. The distinction is clearer in a simple example:

sum( 1/factorial(n) for n in 0:8 )      # generator
2.71827876984127
sum( [1/factorial(n) for n in 0:8] )    # comprehension
2.71827876984127

These statements perform the same computation, but the former just runs a loop, while the second constructs a vector from the loop and then sums the entries of the vector.
